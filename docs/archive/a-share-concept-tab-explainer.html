A 股概念分析 Tab 现在在做什么

当前页面里真正的“概念分析”是独立的“股票概念” Tab：它从某一天已入库的知识星球话题里，提取被明确提到的 A 股、关联概念、来源话题和推荐池命中情况。

入口位置 群组详情页顶部主标签：话题列表 / 文件 / A股分析 / 每日总结 / 股票概念
实现组件 DailyTopicAnalysisPanel 的 mode="stock-concepts"
核心结论 它是同日解释工具，不是多日推荐池本身。

一句话理解

输入

某个群组、某一天的话题

默认按报告日期读取当天已入库话题，不负责重新爬取内容。

处理

AI 提取股票与概念

只保留明确提到或能从上下文明确映射到 A 股上市公司的结果，并记录来源话题。

输出

概念榜、股票表、详情回溯

前端把结果整理成概念聚合、股票列表、7 天趋势、推荐池共振和原始话题弹窗。

数据怎么流动

1. 选日期

报告日期

前端用日期输入框确定要看的交易日或自然日。

2. 创建任务

POST /api/analysis/daily-stock-concepts/{group_id}

后端创建 daily_stock_concepts 后台任务，并把任务 ID 交给已有任务日志浮层。

3. 读取话题

当天已入库话题

服务复用每日总结的日期解析、话题读取和提示词组装逻辑。

4. AI 结构化输出

股票、概念、理由、topic_ids

输出受 JSON Schema 约束；股票名称会再尝试用 stock_basic 补齐代码和市场。

5. 前端展示

概念聚合与回溯

结果存入 daily_stock_concepts，前端再聚合成概念榜、趋势和股票表。

它展示哪些判断

概念榜排序

先按去重来源话题数排序，再看推荐池命中数量、相关股票数，最后按中文概念名排序。

质量标签

多话题共振 推荐池共振 单话题扩散 新出现 持续升温

推荐池命中

前端会用同一天的 A 股分析图表接口取 Top 100 排名，把公司名规范化后与概念结果比对。

详情弹窗

点击股票可看当天概念、来源话题和近 7 天出现情况；点击来源话题 ID 可回到原始话题内容。

和 A 股分析 Tab 的边界

股票概念 Tab

  • 对象：某一天的话题。
  • 目标：解释“今天聊到了哪些股票和概念”。
  • 输出：股票、概念、理由、来源话题、7 天趋势。
  • 存储：daily_stock_concepts。

A 股分析 Tab

  • 对象：多日公司提及结果和排名窗口。
  • 目标：形成公司热度榜、推荐池和通达信导出。
  • 输出：日期序列、排名窗口、公司提及统计、导出板块。
  • 存储：zsxq_a_share_daily_mentions 等表。

使用时要注意

  • 没有当天结果时，页面会提示“还没有股票概念结果”，需要先点“提取”。
  • “未匹配”不一定代表 AI 没识别到股票，而是没有从股票基础库可靠补齐代码。
  • 置信度低的结果通常来自名称不完整、上下文不够确定或代码无法匹配。
  • 推荐池命中只是共振标记，不等于买入建议。
  • 这个 Tab 适合做盘后解释和线索回溯，不应该替代多日推荐池排名。
